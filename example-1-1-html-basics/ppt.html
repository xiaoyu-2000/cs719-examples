Slide 4

Tables

  • Often we are given data that makes sense to present in a tabular fashion
  • Tables are very flexible and have been used for page layout in HTML documents for many years
  • No longer recommended as it distracts from the purpose of a table – displaying tabular data
  • Trend with HTML5 is to avoid structures like tables for layout and instead leave that to CSS
  • More on this in future lectures

表格

  • 我们常常会得到一些数据，这些数据以表格形式展示是很有意义的
  • 表格非常灵活，多年来一直用于 HTML 文档的页面布局
  • 不再推荐使用，因为这会偏离表格的本质——展示表格式数据
  • HTML5 的趋势是避免使用表格布局，而将其留给 CSS
  • 将来讲座中会更多讨论

Slide 5

Tables

  • <table> - Defines an HTML table and groups the components of the table together
  • <tr> - A row in the table that contains a number of cells
  • <td> - A data cell in the table
  • <th> - A header cell in the table
  • <thead> - Table header represents the top-most row of a table
  • <tbody> - Table body, the main content of a table
  • <tfoot> - Table footer represents the bottom-most row of a table
  • <caption> - A caption to attach or associate with the table, must appear immediately after the opening <table> tag

表格

  • <table> - 定义 HTML 表格并将表格组件组合在一起
  • <tr> - 表格中的一行，包含多个单元格
  • <td> - 表格中的数据单元格
  • <th> - 表格中的表头单元格
  • <thead> - 表格头部，表示表格的顶行
  • <tbody> - 表格主体，主要内容
  • <tfoot> - 表格尾部，表示表格的底行
  • <caption> - 表格的标题，必须出现在 <table> 标签之后

Slide 7

Basic table observations

  • The table will have as many rows as there are <tr> elements
  • The table will have as many columns as the largest number of <td> elements inside a <tr>
  • If one row has five cells and any others have 3 cells each the table will have five columns
  • Cells are placed left-to-right

基础表格观察

  • 表格的行数由 <tr> 元素的数量决定
  • 表格的列数由包含最多 <td> 元素的行决定
  • 如果一行有五个单元格，其他行各有三个单元格，则表格有五列
  • 单元格从左到右排列

Slide 16

Intermediate table observations

  • The order of <tbody>, <thead>, and <tfoot> inside the <table> does not matter
  • <thead> will always be rendered as the top-most row in the table above the <tbody>
  • <tfoot> will always be rendered as the bottom-most row in the table below the <tbody>
  • The order of <tr> elements does matter. They will be displayed in the order in which they are declared in their respective sections

中级表格观察

  • <tbody>, <thead>, <tfoot> 在 <table> 内的顺序无关紧要
  • <thead> 总是显示在表格的最顶部，在 <tbody> 上面
  • <tfoot> 总是显示在表格的最底部，在 <tbody> 下面
  • <tr> 元素的顺序很重要，它们会按照在各自部分中声明的顺序显示

Slide 17-18

HTML attributes

  • Attributes are name/value pairs that are seen inside the opening tags of HTML elements. We have seen some of these already
  • All HTML elements can have attributes, some just have more than others
  • Attributes are used to provide additional information about an element and come in 4 major varieties
    • Required – The tag needs these attributes to function correctly
    • Optional – The tag does not require these attributes to function but useful functionality can be added using them
    • Standard – Available to most tags, provide general functionality
    • Event – Will discuss these in future lectures

HTML 属性

  • 属性是 HTML 元素的起始标签内的名称/值对。我们已经看过一些了
  • 所有 HTML 元素都可以有属性，只是有些元素有更多属性
  • 属性用于提供有关元素的附加信息，主要有四种类型
    • 必需 – 标签需要这些属性才能正常工作
    • 可选 – 标签不需要这些属性也能正常工作，但使用它们可以添加有用的功能
    • 标准 – 可用于大多数标签，提供一般功能
    • 事件 – 将在未来的讲座中讨论这些

Slide 22-25

Relative and absolute paths

Example

<a href="A.html">Article</a> or <a href="./A.html">Article</a>

"./" or nothing means relative to the location of the current file; e.g. the file A.html is in the same folder as the current file myPage.html

<a href="../folder2/B.html">Article</a>

"../" means one level up relative to the current location; e.g. the file B.html is one level up and then within the folder2 directory relative to myPage.html

<a href="/folder2/B.html">Article</a>

"/" means relative to the root; e.g. the file B.html is within the folder2 directory that is within the root directory

相对路径和绝对路径

示例

<a href="A.html">Article</a> 或 <a href="./A.html">Article</a>

"./" 或无表示相对于当前文件的位置；例如，文件 A.html 在与当前文件 myPage.html 相同的文件夹中

<a href="../folder2/B.html">Article</a>

"../" 表示相对于当前文件位置向上一级；例如，文件 B.html 位于 myPage.html 上一级目录中的 folder2 文件夹内

<a href="/folder2/B.html">Article</a>

"/" 表示相对于根目录；例如，文件 B.html 位于根目录中的 folder2 文件夹内

Slide 32

Images - File Types

  • Browsers usually support: GIFs, PNGs, JPEGs/JPGs, BMP, and ICO; but not TIFF, JPEG2000, DjVu, etc
  • JPEGs are a lossy format with the quality settings set in software
  • Choose JPEGs for photos and photo-like images
  • GIFs & PNGs are lossless formats, PNG usually produces smaller file sizes
  • Choose GIFs or PNGs for flat color graphics, text, simple logos, and screenshots. Choose PNG if you want transparency
  • GIFs can also be animated, PNG animation (APNG, MNG) is not supported by browsers, so simple image animation on the Web has to be GIF

图像 - 文件类型

  • 浏览器通常支持：GIF、PNG、JPEG/JPG、BMP 和 ICO；但不支持 TIFF、JPEG2000、DjVu 等
  • JPEG 是有损格式，质量设置在软件中进行
  • 为照片和类似照片的图像选择 JPEG
  • GIF 和 PNG 是无损格式，PNG 通常生成较小的文件大小
  • 为平面颜色图形、文本、简单标志和截图选择 GIF 或 PNG。如果需要透明度，选择 PNG
  • GIF 还可以是动画，PNG 动画（APNG、MNG）不受浏览器支持，因此网页上的简单图像动画必须使用 GIF

Images - More points

  • You can resize a large image to fit in a smaller area using the width and height attributes, however this is not always wise
  • You are still downloading a large photo which can be slow
  • Sometimes you are better off creating a smaller "thumbnail" version of the image and linking to the larger version
<a href="fullsize_image.png">
    <img src="thumbnail_image.png">
</a>

图像 - 更多要点

  • 可以使用 width 和 height 属性调整大图像的大小以适应较小的区域，但这并不总是明智的
  • 你仍然在下载大照片，这可能很慢
  • 有时最好创建一个较小的图像缩略图，并链接到较大的版本
<a href="fullsize_image.png">

Forms - Checkboxes

Checkboxes normally appear in a logical grouping in which 0 or more items can be selected. They are created using <input> controls with the type attribute set to checkbox

  • Checkboxes can be grouped together by setting the name attributes of related inputs to the same value

Forms - Submitting

When the user clicks the 'Submit' button, we want to have the form data processed in some way. This is normally done by a piece of program code, e.g., JavaScript code executed in the browser or alternatively by server-side code (e.g., a Servlet or PHP script) executed on a server

We might call this piece of code a form processing agent – and we need to identify the agent that the form data should be sent to

We identify the agent in the action attribute of the <form> element

<form action="https://trex-sandwich.com/echo/">   <!-- form content here --></form>
<form action="../processing_script.jsp">   <!-- form content here --></form>

表单 - 提交

当用户点击“提交”按钮时，我们希望以某种方式处理表单数据。这通常由一段程序代码完成，例如，在浏览器中执行的 JavaScript 代码或在服务器上执行的服务器端代码（例如 Servlet 或 PHP 脚本）

我们可能会将这段代码称为表单处理代理，并且需要确定表单数据应发送到哪个代理

我们在 <form> 元素的 action 属性中标识代理

<form action="https://trex-sandwich.com/echo/">   <!-- 表单内容 --></form>
<form action="../processing_script.jsp">   <!-- 表单内容 --></form>

Form - Methods

Along with the action attribute, a form element has a method attribute which defines the HTTP method (verb) to use when communicating with the server

For forms, the verb is normally GET or POST

The method chosen affects the HTTP message header and determines the way in which the data will be transported to the server

The HTTP message header is a bit out of scope for now, but the way in which the data is transported is relevant to us

表单 - 方法

除了 action 属性，表单元素还有一个 method 属性，用于定义与服务器通信时使用的 HTTP 方法（动词）

对于表单，动词通常是 GET 或 POST

选择的方法会影响 HTTP 消息头，并决定数据传输到服务器的方式

HTTP 消息头暂时超出我们的讨论范围，但数据传输方式与我们相关

Slide 72

GET vs POST

  • The browser is responsible for sending data from a form to the destination specified in the action attribute
  • If the GET method is used, name=value pairs are appended to the action attribute address starting with a "?" separated by "&"
  • Example: https://ex.com/form.jsp?fname=Peter&lname=Jackson&gend=male
  • If the POST method is used, the data is sent as the body of a message
  • Not visible in the browser/history. Useful for passwords!

GET 与 POST 的区别

  • 浏览器负责将表单数据发送到 action 属性指定的目的地
  • 如果使用 GET 方法，name=value 对附加到 action 属性地址，起始于 "?"，用 "&" 分隔
  • 示例：https://ex.com/form.jsp?fname=Peter&lname=Jackson&gend=male
  • 如果使用 POST 方法，数据作为消息主体发送
  • 在浏览器/历史记录中不可见。适用于密码等敏感信息！
